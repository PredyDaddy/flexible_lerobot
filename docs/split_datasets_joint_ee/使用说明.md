# 数据切分脚本使用说明（Joint / EE）

脚本路径：`docs/split_datasets_joint_ee/split_ee_or_joint_dataset.py`

兼容入口（包装脚本）：`scripts/split_ee_or_joint_dataset.py`（参数完全一致）

---

## 1. 目的

把 LeRobot Dataset v3.0（AgileX 录制的数据）按低维状态拆成两份数据集：

- `joint`：仅保留关节角（14 维：左右臂各 7），用于关节空间训练/推理。
- `ee`：仅保留末端位姿（12 维：左右臂各 6），用于末端空间训练/推理。

输出的数据集仍然是 **v3.0 标准目录结构**（包含 `meta/ data/ videos/`），可直接用 `LeRobotDataset(root=...)` 读取。

---

## 2. 环境要求

- 使用 `my_lerobot` conda 环境运行（仓库约定）
- 数据集为 v3.0 结构：目录内包含 `meta/info.json`、`meta/episodes/`、`data/`、`videos/` 等
- 如果要生成 `ee` 切分，原数据集的 `observation.state` 必须包含 `left_ee.*` / `right_ee.*` 字段（即录制时 `record_ee_pose=true`）

---

## 3. 基本用法

以你的数据为例（原始数据根目录）：

`/home/agilex/.cache/huggingface/lerobot/cqy/agilex_vla_demo_ee_pinocchio_Bowl_on_Plate_Placement_test`

### 3.1 同时生成 joint + ee（推荐）

在仓库根目录执行：

```bash
python docs/split_datasets_joint_ee/split_ee_or_joint_dataset.py \
  --input-root /home/agilex/.cache/huggingface/lerobot/cqy/agilex_vla_demo_ee_pinocchio_Bowl_on_Plate_Placement_test \
  --spaces both \
  --video-mode auto \
  --overwrite
```

默认输出到仓库下：`outputs/split_datasets/`

会生成：

- `outputs/split_datasets/agilex_vla_demo_ee_pinocchio_Bowl_on_Plate_Placement_test_joint`
- `outputs/split_datasets/agilex_vla_demo_ee_pinocchio_Bowl_on_Plate_Placement_test_ee`

### 3.2 只生成其中一种

```bash
python docs/split_datasets_joint_ee/split_ee_or_joint_dataset.py --input-root <DATASET_ROOT> --spaces joint
python docs/split_datasets_joint_ee/split_ee_or_joint_dataset.py --input-root <DATASET_ROOT> --spaces ee
```

### 3.3 指定输出目录

```bash
python docs/split_datasets_joint_ee/split_ee_or_joint_dataset.py \
  --input-root <DATASET_ROOT> \
  --output-base /tmp/split_outputs \
  --spaces both \
  --overwrite
```

---

## 4. 参数说明

- `--input-root`：原始数据集根目录（包含 `meta/ data/ videos/`）
- `--output-base`：输出基目录（默认：`<repo_root>/outputs/split_datasets`）
- `--spaces`：生成哪些切分
  - `joint` / `ee` / `both`
- `--video-mode`：视频复制方式
  - `auto`：优先 hardlink，失败则 symlink，再失败则 copy
  - `hardlink`：省空间，但要求同一文件系统
  - `symlink`：省空间，但移动/打包时要注意链接
  - `copy`：最稳，但占空间
- `--overwrite`：输出目录存在时覆盖

---

## 5. 切分后数据的语义（简述）

### 5.1 joint 版本

- `observation.state`：从原 `observation.state` 中按 `meta/info.json -> features["action"]["names"]` 对齐抽取关节维度
- `action`：保持原样（关节动作，14 维）

### 5.2 ee 版本

- `observation.state`：从原 `observation.state` 中抽取 `left_ee.*` + `right_ee.*`（共 12 维）
- `action`：用同一 episode 内的 **下一帧 EE state** 做 shift：
  - `action[t] = observation.state[t+1]`
  - episode 边界与 episode 最后一帧：`action[t] = observation.state[t]`（避免跨 episode 串联）

## 6. 读取验证（可选）

如果你在 `LeRobotDataset` 读取时遇到 HF datasets cache 的锁文件权限问题，可临时指定可写 cache：

```bash
HF_DATASETS_CACHE=/tmp/hf_datasets_cache python - <<'PY'
from lerobot.datasets.lerobot_dataset import LeRobotDataset

ds_joint = LeRobotDataset(repo_id="local", root="outputs/split_datasets/agilex_vla_demo_ee_pinocchio_Bowl_on_Plate_Placement_test_joint")
ds_ee = LeRobotDataset(repo_id="local", root="outputs/split_datasets/agilex_vla_demo_ee_pinocchio_Bowl_on_Plate_Placement_test_ee")

print(ds_joint[0]["observation.state"].shape, ds_joint[0]["action"].shape)
print(ds_ee[0]["observation.state"].shape, ds_ee[0]["action"].shape)
PY
```
