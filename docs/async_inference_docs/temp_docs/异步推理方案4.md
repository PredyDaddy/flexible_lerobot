# 异步推理方案 4: 双臂独立异步推理 (AgileX 定制版)

本文档描述了如何利用 `lerobot.async_inference` 模块，将 AgileX 双臂机器人的推理过程解耦为两个独立的异步服务。
这种方案可以显著提高控制频率（FPS），减少由于模型推理阻塞导致的控制延迟。

## 架构概览

我们将系统分为 4 个独立的进程：

1.  **左臂策略服务器 (Policy Server Left):** 负责加载左臂模型，接收左臂观测，返回左臂动作。
2.  **右臂策略服务器 (Policy Server Right):** 负责加载右臂模型，接收右臂观测，返回右臂动作。
3.  **左臂机器人客户端 (Robot Client Left):** 控制左臂，采集图像，发送给左臂服务器，执行返回的动作。
4.  **右臂机器人客户端 (Robot Client Right):** 控制右臂，采集图像，发送给右臂服务器，执行返回的动作。

每个客户端将独立运行，互不干扰（除了在硬件层面共享同一个底盘/控制器，但这由 ROS Bridge 处理）。

## 1. 启动策略服务器

你需要打开两个终端，分别启动左臂和右臂的推理服务。

**终端 1: 左臂服务器 (端口 8080)**

```bash
# 请替换为你实际的左臂 checkpoint 路径
export LEFT_CHECKPOINT="outputs/act_agilex_left_box/checkpoints/030000/pretrained_model"

python -m lerobot.async_inference.policy_server \
    --host 0.0.0.0 \
    --port 8080 \
    --fps 30 \
    --policy.type act \
    --policy.pretrained_name_or_path "$LEFT_CHECKPOINT" \
    --policy.actions_per_chunk 50 \
    --policy.device cuda \
    --policy.rename_map '{"camera_left": "camera_left", "camera_front": "camera_front"}'
```

**终端 2: 右臂服务器 (端口 8081)**

```bash
# 请替换为你实际的右臂 checkpoint 路径
export RIGHT_CHECKPOINT="outputs/act_agilex_right_black_cup/checkpoints/last/pretrained_model"

python -m lerobot.async_inference.policy_server \
    --host 0.0.0.0 \
    --port 8081 \
    --fps 30 \
    --policy.type act \
    --policy.pretrained_name_or_path "$RIGHT_CHECKPOINT" \
    --policy.actions_per_chunk 50 \
    --policy.device cuda \
    --policy.rename_map '{"camera_right": "camera_right", "camera_front": "camera_front"}'
```

> **注意:** `policy.rename_map` 参数确保客户端发送的相机名称能正确映射到模型期望的名称。如果模型训练时使用了特定的键名，请在此处调整。

## 2. 启动机器人客户端

你需要使用定制的客户端脚本 `agilex_infer_async_client.py`，它结合了 `agilex_infer_single_cc_vertical.py` 的单臂控制逻辑和 `robot_client.py` 的异步通信能力。

**终端 3: 左臂客户端 (连接 8080)**

```bash
python agilex_infer_async_client.py \
    --arm left \
    --server_address localhost:8080 \
    --fps 30 \
    --binary-gripper \
    --gripper-threshold 0.7
```

**终端 4: 右臂客户端 (连接 8081)**

```bash
python agilex_infer_async_client.py \
    --arm right \
    --server_address localhost:8081 \
    --fps 30 \
    --binary-gripper \
    --gripper-threshold 0.7
```

## 3. 定制客户端脚本说明 (`agilex_infer_async_client.py`)

该脚本是为了适配 AgileX 双臂机器人的特殊需求（单臂独立控制、保持另一臂静止、特定相机配置）而开发的。

### 主要特性
- **单臂专注:** 初始化时只连接当前臂需要的相机（减少带宽）。
- **安全控制:** 使用 `send_single_arm_action` 逻辑，在控制一侧臂时，主动读取另一侧臂的位置并保持不变，防止“掉臂”。
- **夹爪处理:** 支持 `--binary-gripper` 参数，在客户端进行夹爪动作的二值化处理。
- **异步通信:** 继承自 `lerobot` 的异步架构，推理和控制循环分离。

### 脚本位置
该脚本应放置在项目根目录下 (即与 `agilex_infer_single_cc_vertical.py` 同级)。

## 4. 性能优化建议

1.  **网络延迟:** 尽量在同一台机器上运行服务器和客户端（localhost），或者使用千兆/万兆局域网。
2.  **图像压缩:** 目前 `lerobot` 默认通过 gRPC 发送原始图像数据。如果网络是瓶颈，可能需要修改 `policy_server` 和 `robot_client` 以支持 JPEG 压缩。
3.  **多 GPU:** 如果有多个 GPU，可以在启动 `policy_server` 时通过 `CUDA_VISIBLE_DEVICES` 指定不同的 GPU，例如左臂用 GPU 0，右臂用 GPU 1。

```bash
# 左臂
CUDA_VISIBLE_DEVICES=0 python -m lerobot.async_inference.policy_server ...
# 右臂
CUDA_VISIBLE_DEVICES=1 python -m lerobot.async_inference.policy_server ...
```
