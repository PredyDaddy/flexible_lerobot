## 1. policies/ - 策略算法

策略模块是 LeRobot 的核心，实现了多种模仿学习和强化学习算法。

### 1.1 模块结构

```
policies/
├── __init__.py              # 导出所有策略配置
├── act/                     # ACT 策略
│   ├── configuration_act.py
│   └── modeling_act.py
├── diffusion/               # Diffusion 策略
│   ├── configuration_diffusion.py
│   └── modeling_diffusion.py
├── tdmpc/                   # TD-MPC 策略
│   ├── configuration_tdmpc.py
│   ├── modeling_tdmpc.py
│   └── processor_tdmpc.py
├── vqbet/                   # VQ-BeT 策略
│   ├── configuration_vqbet.py
│   ├── modeling_vqbet.py
│   └── vqbet_utils.py
├── smolvla/                 # SmolVLA 视觉语言动作模型
│   ├── configuration_smolvla.py
│   ├── modeling_smolvla.py
│   └── processor_smolvla.py
├── pi0/                     # PI0 策略
├── pi05/                    # PI0.5 策略
├── groot/                   # Gr00t 策略
├── sac/                     # SAC 强化学习策略
│   ├── configuration_sac.py
│   ├── modeling_sac.py
│   └── reward_model/        # 奖励模型
└── rtc/                     # RTC 实时控制策略
    ├── modeling_rtc.py
    ├── action_queue.py
    └── latency_tracker.py
```

### 1.2 策略类型说明

| 策略 | 类型 | 说明 |
|------|------|------|
| **ACT** | 模仿学习 | Action Chunking with Transformers，使用 Transformer 预测动作序列 |
| **Diffusion** | 模仿学习 | 基于扩散模型的策略，通过去噪过程生成动作 |
| **TD-MPC** | 模型预测控制 | 时序差分模型预测控制，结合学习和规划 |
| **VQ-BeT** | 模仿学习 | 向量量化行为 Transformer，离散化动作空间 |
| **SmolVLA** | VLA | 小型视觉-语言-动作模型，支持语言指令 |
| **PI0/PI0.5** | VLA | Physical Intelligence 的视觉语言动作模型 |
| **Gr00t** | VLA | NVIDIA 的通用机器人策略 |
| **SAC** | 强化学习 | Soft Actor-Critic，用于在线学习 |

### 1.3 策略文件命名规范

每个策略子目录遵循统一的命名规范：
- `configuration_*.py`: 策略配置类，定义超参数
- `modeling_*.py`: 策略模型实现，包含网络结构和前向传播
- `processor_*.py`: 数据预处理/后处理（可选）

### 1.4 核心接口

所有策略都需要实现以下核心方法：

```python
class Policy:
    def forward(self, batch: dict) -> dict:
        """训练时的前向传播，返回损失"""
        pass

    def select_action(self, batch: dict) -> Tensor:
        """推理时选择动作"""
        pass
```

### 2.2 LeRobotDataset 核心类

`LeRobotDataset` 是数据集的核心类，继承自 `torch.utils.data.Dataset`：

```python
class LeRobotDatasetMetadata:
    """数据集元数据管理"""
    repo_id: str           # HuggingFace 仓库 ID

    revision: str          # 版本号 (默认 v3.0)

class LeRobotDataset(torch.utils.data.Dataset):
    """主数据集类"""
    # 核心功能：
    # - 从 HuggingFace Hub 或本地加载数据
```

### 2.3 数据格式

- **元数据**: Parquet 文件存储 episode 信息

### 3.2 Robot 抽象基类

所有机器人必须继承 `Robot` 基类并实现以下接口：

```python
class Robot(abc.ABC):
    """机器人抽象基类"""

    # 必须在子类中设置
    config_class: type[RobotConfig]
    name: str

    @property
    @abstractmethod
    def observation_features(self) -> dict:
        """观测空间描述"""
        pass

    @property
    @abstractmethod
    def action_features(self) -> dict:
        """动作空间描述"""
        pass

    @abstractmethod
    def connect(self, calibrate: bool = True) -> None:
        """连接机器人"""
        pass

    @abstractmethod
    def get_observation(self) -> dict[str, Any]:
        """获取当前观测"""
        pass

    @abstractmethod
    def send_action(self, action: dict[str, Any]) -> dict[str, Any]:
        """发送动作指令"""
        pass

    @abstractmethod
    def disconnect(self) -> None:
        """断开连接"""
        pass
```

### 4.2 Camera 抽象基类

```python
class Camera(abc.ABC):
    """相机抽象基类"""

    def __init__(self, config: CameraConfig):
        self.fps: int | None = config.fps
        self.width: int | None = config.width
        self.height: int | None = config.height

    @staticmethod
    @abstractmethod
    def find_cameras() -> list[dict[str, Any]]:
        """检测可用相机"""
        pass

    @abstractmethod
    def connect(self, warmup: bool = True) -> None:
        """连接相机"""
        pass

    @abstractmethod
    def read(self, color_mode: ColorMode | None = None) -> NDArray:
        """同步读取一帧"""
        pass

    @abstractmethod
    def async_read(self, timeout_ms: float) -> NDArray:
        """异步读取一帧"""
        pass

    @abstractmethod
    def disconnect(self) -> None:
        """断开连接"""
        pass
```

## 典型工作流

### 1. 数据采集流程

```
teleoperator.get_action() → robot.send_action() → camera.read()
                                    ↓
                            dataset.add_frame()
```

### 2. 训练流程

```
dataset → dataloader → policy.forward() → loss.backward() → optimizer.step()
```

### 3. 部署流程

```
camera.read() → processor → policy.select_action() → robot.send_action()
```
