## 1. 什么是运动学

### 1.1 运动学的定义

**运动学 (Kinematics)** 研究的是机器人各部件之间的几何关系，不考虑力和力矩。

简单来说：
- **正运动学 (FK)**: 已知关节角度 → 求末端位置
- **逆运动学 (IK)**: 已知末端位置 → 求关节角度

```
┌─────────────────────────────────────────────────────────┐
│                                                         │
│   关节角度 [θ1, θ2, θ3, θ4, θ5, θ6]                    │
│              │                    ▲                     │
│              │ FK                 │ IK                  │
│              ▼                    │                     │
│   末端位姿 [x, y, z, roll, pitch, yaw]                 │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 为什么需要运动学

| 场景 | 需要的运动学 | 说明 |
|------|-------------|------|
| 录制数据时记录末端位置 | FK | 从关节角度计算末端在哪 |
| 回放时控制末端到目标位置 | IK | 从目标位置计算关节角度 |
| 可视化机器人姿态 | FK | 显示机器人当前状态 |
| 任务空间控制 | IK | 直接控制末端移动 |

### 1.3 本项目的应用

在 LeRobot 项目中：
- **录制 (record_ee_pinocchio.sh)**: 使用 FK 计算末端位姿，存入数据集
- **回放 (replay_ee_pinocchio.sh)**: 使用 IK 从末端位姿计算关节角度，控制机器人

## 2. 正运动学 (FK) 详解

### 2.1 基本原理

正运动学就是一系列**坐标变换的链式乘法**。

想象一个 6 轴机械臂：
```
基座 → 关节1 → 关节2 → 关节3 → 关节4 → 关节5 → 关节6 → 末端
  │       │       │       │       │       │       │       │
  T0      T1      T2      T3      T4      T5      T6     T_ee
```

末端相对于基座的位姿：
```
T_base_to_ee = T0 × T1(θ1) × T2(θ2) × T3(θ3) × T4(θ4) × T5(θ5) × T6(θ6)
```

其中 `Ti(θi)` 是第 i 个关节的变换矩阵，取决于关节角度 θi。

### 2.2 齐次变换矩阵

每个变换用 **4×4 齐次变换矩阵** 表示：

```
T = | R  t |    R: 3×3 旋转矩阵
    | 0  1 |    t: 3×1 平移向量

例如:
T = | r11 r12 r13 tx |
    | r21 r22 r23 ty |
    | r31 r32 r33 tz |
    |  0   0   0   1 |
```

**为什么用 4×4 矩阵？**
- 可以同时表示旋转和平移
- 多个变换可以直接矩阵相乘

### 2.3 代码中的 FK 实现

```python
# 文件: my_ik/piper_pinocchio_kinematics.py

    """
    # 1. 计算正运动学 (Pinocchio 内部做了链式乘法)
    pin.forwardKinematics(self.reduced_robot.model, self.reduced_robot.data, q)
    pin.updateFramePlacements(self.reduced_robot.model, self.reduced_robot.data)

    ee_pose = self.reduced_robot.data.oMf[self.ee_frame_id]

    return _matrix_to_xyzrpy(T)
```

### 2.4 FK 的特点

| 特点 | 说明 |
|------|------|
| **唯一解** | 给定关节角度，末端位置唯一确定 |
| **计算简单** | 只是矩阵乘法，计算量小 |
| **总是成功** | 不存在无解的情况 |

## 3. 逆运动学 (IK) 详解

### 3.1 基本原理

逆运动学是 FK 的反问题：**已知末端位姿，求关节角度**。

```
已知: T_target (目标末端位姿)
求解: [θ1, θ2, θ3, θ4, θ5, θ6] 使得 FK(θ) ≈ T_target
```

### 3.2 IK 的难点

| 难点 | 说明 |
|------|------|
| **多解** | 同一末端位置可能有多组关节角度 |
| **无解** | 目标位置可能超出工作空间 |
| **奇异点** | 某些姿态下雅可比矩阵奇异 |
| **非线性** | 无法直接求解，需要迭代优化 |

### 3.3 IK 求解方法

本项目使用 **数值优化方法** (Casadi + IPOPT)：

```
最小化: ||FK(θ) - T_target||² + λ||θ||²
约束:   θ_min ≤ θ ≤ θ_max  (关节限位)
```

- 第一项：位姿误差（越小越好）
- 第二项：正则化（防止关节角度过大）
- 约束：关节不能超出物理限位

### 3.4 代码中的 IK 实现

```python
# 文件: my_ik/piper_pinocchio_kinematics.py

def _setup_casadi_solver(self):
    """设置优化问题"""
    # 1. 定义优化变量 (6个关节角度)
    self.var_q = self.opti.variable(self.nq)

    # 2. 定义目标函数
    # 位姿误差 (使用 log6 计算 SE3 误差)
    totalcost = casadi.sumsqr(self.error(self.var_q, self.param_tf))
    # 正则化项
    regularization = casadi.sumsqr(self.var_q)
    # 总代价
    self.opti.minimize(
        self.pose_weight * totalcost +           # 位姿误差权重
        self.regularization_weight * regularization  # 正则化权重
    )

    # 3. 添加关节限位约束
    self.opti.subject_to(
        self.opti.bounded(
            self.reduced_robot.model.lowerPositionLimit,  # 下限
            self.var_q,
            self.reduced_robot.model.upperPositionLimit,  # 上限
        )
    )
```

### 3.5 IK 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `max_iter` | 50 | 最大迭代次数 |
| `tol` | 1e-4 | 收敛容差 |
| `pose_weight` | 20.0 | 位姿误差权重 |
| `regularization_weight` | 0.01 | 正则化权重 |

## 4. 坐标系与位姿表示

### 4.1 位姿的两种表示

**方式一：4×4 齐次变换矩阵**
```
T = | R  t |   包含旋转(R)和平移(t)
    | 0  1 |   共16个数，但只有6个自由度
```

**方式二：xyzrpy (6维向量)**
```
[x, y, z, roll, pitch, yaw]
 位置(米)    姿态(弧度)
```

### 4.2 欧拉角 (Roll-Pitch-Yaw)

```
Roll  (绕X轴): 翻滚
Pitch (绕Y轴): 俯仰
Yaw   (绕Z轴): 偏航

旋转顺序: R = Rz(yaw) × Ry(pitch) × Rx(roll)
```

### 4.3 代码中的转换函数

```python
# 矩阵 → xyzrpy
def _matrix_to_xyzrpy(matrix: np.ndarray) -> np.ndarray:
    x, y, z = matrix[0, 3], matrix[1, 3], matrix[2, 3]
    R = matrix[:3, :3]
    roll = math.atan2(R[2, 1], R[2, 2])
    pitch = math.asin(-R[2, 0])
    yaw = math.atan2(R[1, 0], R[0, 0])
    return np.array([x, y, z, roll, pitch, yaw])
```

**IK 求解流程：**

```python
def inverse_kinematics(self, target_xyzrpy, q_init=None):
    # 1. 转换目标位姿为 4×4 矩阵
    target_matrix = _xyzrpy_to_matrix(target)

    # 2. 设置初始猜测 (热启动)
    self.opti.set_initial(self.var_q, self.init_data)

    # 3. 设置目标
    self.opti.set_value(self.param_tf, target_matrix)

    # 4. 求解
    sol = self.opti.solve_limited()
    sol_q = np.array(self.opti.value(self.var_q)).flatten()

    # 5. 检查解的跳变 (防止不连续)
    max_diff = np.max(np.abs(self.history_data - sol_q))
    if max_diff > 30° (弧度):
        self.init_data = np.zeros(self.nq)  # 重置
    else:
        self.init_data = sol_q.copy()       # 用于下次热启动

    return sol_q, success
```

### 5.4 关键技术点

#### 5.4.1 SE3 误差计算 (log6)

```python
# 位姿误差使用 log6 计算
error = cpin.log6(
    self.cdata.oMf[self.ee_frame_id].inverse() * cpin.SE3(self.cTf)
).vector
```

**为什么用 log6？**
- 直接比较两个 4×4 矩阵不合理（旋转和平移单位不同）
- `log6` 将 SE3 误差映射到 6 维向量（3 平移 + 3 旋转）
- 这是李代数的概念，保证了误差度量的合理性

#### 5.4.2 热启动 (Warm Start)

```python
# 使用上一次的解作为初始猜测
self.opti.set_initial(self.var_q, self.init_data)
```

**好处：**
- 加速收敛（从接近解的地方开始）
- 保证解的连续性（避免跳变）

#### 5.4.3 跳变检测

```python
max_diff = np.max(np.abs(self.history_data - sol_q))
if max_diff > 30.0 / 180.0 * np.pi:  # 30度
    self.init_data = np.zeros(self.nq)  # 重置
```

**为什么需要？**
- IK 有多解，优化器可能跳到另一个解
- 跳变会导致机器人运动不连续
- 检测到跳变后重置，让下次从零开始

### 6.4 7 轴 IK 的特殊考虑

7 轴机械臂比 6 轴多一个自由度，这带来了新的问题：

#### 6.4.1 冗余自由度

```
6 轴: 末端位姿有 6 个自由度，关节也是 6 个 → 刚好匹配
7 轴: 末端位姿有 6 个自由度，关节有 7 个 → 多出 1 个自由度
```

**影响：**
- IK 有无穷多解（不只是有限多解）
- 需要额外的约束来选择"最优"解

#### 6.4.2 处理冗余的方法

**方法 1：增加正则化权重**

```python
# 增大正则化权重，让关节角度尽量小
self.opti.minimize(
    pose_weight * totalcost +
    0.1 * regularization  # 增大权重
)
```

**方法 2：添加零空间目标**

```python
# 添加额外目标：让某个关节保持在特定位置
null_space_cost = casadi.sumsqr(self.var_q[3] - 0.0)  # 第4关节保持在0
self.opti.minimize(
    pose_weight * totalcost +
    regularization_weight * regularization +
    0.5 * null_space_cost  # 零空间目标
)
```


### 7.4 总结

| 概念 | 要点 |
|------|------|
| **FK** | 关节角度 → 末端位姿，唯一解，计算简单 |
| **IK** | 末端位姿 → 关节角度，可能多解/无解，需要优化 |
| **URDF** | 机器人模型描述文件，包含关节和连杆信息 |
| **Pinocchio** | 高效的机器人运动学库 |
| **Casadi** | 符号计算和优化库，用于 IK 求解 |

**复用到其他机器人的关键步骤：**
1. 准备 URDF 文件
2. 修改 `lock_joints` 参数
3. 修改末端关节名称
4. 调整 `ee_rotation_rpy` 和 `ee_translation`
