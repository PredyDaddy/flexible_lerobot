## 1. 背景与动机

### 1.1 为什么要导出模型？

PyTorch 推理存在以下问题：
- **依赖重**：需要完整的 PyTorch + torchvision 环境
- **启动慢**：模型加载和首次推理需要 JIT 编译
- **部署难**：在嵌入式设备或生产环境中部署 PyTorch 较为复杂

ONNX + ONNXRuntime 的优势：
- **轻量级**：只需 `onnxruntime` + `numpy` + `safetensors`
- **跨平台**：支持 CPU/GPU/TensorRT/OpenVINO 等多种后端
- **易部署**：可进一步转换为 TensorRT Engine 获得更高性能

### 1.2 导出边界选择

ACT 推理链路如下：

```
原始观测 → 预处理 → 归一化 → ACT网络前向 → 动作队列 → 反归一化 → 机器人动作
           ↑                    ↑              ↑
        Python侧            导出这部分       Python侧
```

**我们只导出 ACT 核心网络（`policy.model`）**，原因：
- `ACTPolicy.select_action` 包含 Python 队列（deque），ONNX 无法表达
- 预处理/后处理逻辑简单，用 numpy 实现更灵活
- 核心网络是计算瓶颈，导出它能获得最大收益

## 3. 为什么这样做是可行的

### 3.1 数学等价性

导出的 ONNX 模型与 PyTorch 模型在数学上完全等价：

| 环节 | PyTorch | ONNX + numpy |
|------|---------|--------------|
| 归一化 | `(x - mean) / (std + eps)` | `(x - mean) / (std + eps)` |
| 网络前向 | `policy.model(batch)` | `session.run(inputs)` |
| 反归一化 | `x * std + mean` | `x * std + mean` |

导出脚本内置了验证机制，确保 PyTorch 输出与 ONNX 输出的 `max_abs_diff < 1e-4`。

### 3.2 关键对齐点

**1. 相机顺序一致**

ACT 模型对相机顺序敏感。导出时会记录相机顺序到 `export_metadata.json`：

```json
{
  "camera_order_visual_keys": [
    "observation.images.camera_left",
    "observation.images.camera_front"
  ]
}
```

推理时必须按相同顺序传入图像。

**2. 归一化参数一致**

使用 checkpoint 中保存的 stats 文件：
- 输入归一化：`policy_preprocessor_step_3_normalizer_processor.safetensors`
- 输出反归一化：`policy_postprocessor_step_0_unnormalizer_processor.safetensors`

**3. 队列语义一致**

复刻 `ACTPolicy.select_action` 的队列逻辑：
- 队列为空时：跑一次前向，输出 chunk `(1, chunk_size, action_dim)`
- 队列非空时：直接取下一步 action，不跑前向

### 3.3 推理时 VAE 行为

虽然 `config.json` 中 `use_vae: true`，但推理时：
- 模型处于 `eval()` 模式
- batch 中不包含 `action`（训练目标）

因此会走确定性分支：`latent_sample = zeros([B, latent_dim])`，无随机性。

## 4. 核心原理详解

### 4.1 ONNX 导出原理

`export_single.py` 的核心逻辑：

```python
# 1. 加载 PyTorch 模型
policy = ACTPolicy.from_pretrained(checkpoint)

# 2. 创建 ONNX-friendly wrapper（展平输入）
class ActChunkOnnxWrapper(nn.Module):
    def forward(self, obs_state_norm, img0_norm, img1_norm):
        batch = {
            "observation.state": obs_state_norm,
            "observation.images": [img0_norm, img1_norm],
        }
        actions, _ = self.act(batch)
        return actions

wrapper = ActChunkOnnxWrapper(policy.model)

# 3. 导出 ONNX
torch.onnx.export(
    wrapper,
    (obs_state_norm, img0_norm, img1_norm),
    "act_single.onnx",
    input_names=["obs_state_norm", "img0_norm", "img1_norm"],
    output_names=["actions_norm"],
    opset_version=17,
)
```

### 4.2 ONNXRuntime 推理原理

`ort_policy.py` 的核心逻辑：

```python
class ActOrtPolicy:
    def __init__(self, checkpoint, onnx_path):
        # 1. 加载 ONNX session
        self.session = ort.InferenceSession(onnx_path)

        # 2. 加载归一化 stats
        self._state_stats = load_mean_std(pre_stats, "observation.state")
        self._action_stats = load_mean_std(post_stats, "action")

    def select_action(self, observation):
        # 3. 队列为空时跑前向
        if self._queue_empty():
            # 预处理
            inputs = self._preprocess(observation)
            # ONNX 前向
            actions_norm = self.session.run(["actions_norm"], inputs)[0]
            # 填充队列
            self._fill_queue(actions_norm)

        # 4. 从队列取 action 并反归一化
        action_norm = self._pop_from_queue()
        action = action_norm * self._action_stats.std + self._action_stats.mean
        return action
```

### 4.3 归一化公式

**输入归一化（MEAN_STD 模式）：**
```
x_norm = (x - mean) / (std + eps)
```
其中 `eps = 1e-8`，防止除零。

**输出反归一化：**
```
action = action_norm * std + mean
```
注意：反归一化不需要 eps。

## 5. 快速上手指南

### 5.1 环境准备

```bash
# 基础依赖
pip install onnx onnxruntime safetensors numpy

# 可选：ONNX 简化
pip install onnxsim

# 可选：GPU 推理
pip install onnxruntime-gpu
```

### 5.2 步骤一：导出 ONNX

```bash
# 基本用法
python trt_act/export/export_single.py \
    --checkpoint outputs/act_agilex_left_box/checkpoints/last/pretrained_model

# 指定输出路径
python trt_act/export/export_single.py \
    --checkpoint outputs/act_agilex_left_box/checkpoints/last/pretrained_model \
    --output ./exported_models/left_arm.onnx

# 完整参数
python trt_act/export/export_single.py \
    --checkpoint <ckpt> \
    --output <path.onnx> \
    --opset 17 \
    --device cpu \
    --verify \
    --simplify
```

导出后会生成：
- `act_single.onnx` - ONNX 模型
- `act_single.onnx.data` - 外部权重（模型较大时）
- `export_metadata.json` - 导出元数据

### 5.3 步骤二：验证导出结果

导出脚本默认会验证，也可以手动检查 `export_metadata.json`：

```json
{
  "verification": {
    "enabled": true,
    "passed": true,
    "max_abs_diff": 1.2e-05,
    "mean_abs_diff": 3.4e-07
  }
}
```

如果 `max_abs_diff > 1e-4`，说明导出有问题。

### 5.4 步骤三：运行 ORT 推理

```bash
python trt_act/inference/agilex_infer_ort_single.py \
    --checkpoint outputs/act_agilex_left_box/checkpoints/last/pretrained_model \
    --arm left \
    --fps 30 \
    --binary-gripper \
    --onnx trt_act/export/exported_models/left_arm.onnx \
    --export-metadata trt_act/export/exported_models/export_metadata.json
```

### 5.5 对比原 PyTorch 推理

| 项目 | PyTorch 推理 | ORT 推理 |
|------|-------------|----------|
| 脚本 | `agilex_infer_single_cc_vertical.py` | `agilex_infer_ort_single.py` |
| 依赖 | torch, torchvision, lerobot | onnxruntime, numpy, safetensors |
| 模型 | `model.safetensors` | `act_single.onnx` |
| 前向 | `policy.select_action()` | `session.run()` |

## 6. 常见问题与排障

### 6.1 导出失败

**问题：** `RuntimeError: Unsupported op`

**解决：**
1. 尝试不同的 opset 版本：`--opset 14` 或 `--opset 18`
2. 尝试切换导出器：`--dynamo` 或不加 `--dynamo`
3. 检查 PyTorch/torchvision 版本兼容性

### 6.2 验证不通过

**问题：** `max_abs_diff` 很大

**排查：**
1. 检查相机顺序是否正确
2. 检查预处理是否一致（/255、CHW、mean/std）
3. 检查 eps 值是否一致

### 6.3 推理结果异常

**问题：** 机器人动作明显跑偏

**排查：**
1. 确认 `export_metadata.json` 中的 `camera_order_visual_keys` 与实际传入顺序一致
2. 确认使用了正确的 stats 文件
3. 用离线数据对比 PyTorch 和 ORT 输出

### 6.4 ONNX 文件过大

**问题：** 导出的 ONNX 文件很大（>500MB）

**原因：** ACT 模型中 `list(tensor)` 展开导致图膨胀

**解决：**
1. 使用 `--simplify` 启用 onnxsim 简化
2. 如果仍然很大，可以考虑修改模型代码（将 list 展开改为纯 tensor 操作）
