## 5. 你特别关心项：language 统计差异解释

### 5.1 事实对比

- A：
  - `info.features` 包含 `language_instruction`
  - `stats.json` 不包含 `language_instruction`
- B：
  - `info.features` 本身不包含 `language_instruction`
  - `stats.json` 也不包含 `language_instruction`

### 5.2 结论

A 的“language 统计缺失”来自于：

1. A 里确实定义了 string feature：`language_instruction`；
2. 当前统计文件没有记录该 string feature 的统计项；
3. 检查脚本会按“features 中存在但 stats 中缺失”给出警告。

这属于 **一致性告警**，不是结构性致命错误，不会导致索引崩溃。

### 5.3 与任务映射一致性

A 的语言字段与任务索引映射是正确的：

- `tasks.index = ['picking bottle', 'picking bottle1']`
- `tasks.task_index = [0, 1]`
- `data.task_index` 仅出现 `{0,1}`
- `language_instruction` 分布：
  - `picking bottle`: `900` 帧
  - `picking bottle1`: `600` 帧
- episode 到 task 映射：
  - ep0/1/2 -> task 0
  - ep3/4 -> task 1

## 10. 训练出问题时的优先排查清单

若后续训练或推理出现异常，建议按顺序排查：

1. **特征键匹配**
   - 核对模型配置中的视觉 key 是否是 A 的：
   - `observation.images.cam_high`, `observation.images.cam_left_wrist`, `observation.images.cam_right_wrist`
2. **任务/语言输入路径**
   - 若模型依赖 `language_instruction`，确认数据 pipeline 是否读取该字段；
   - 若模型只用 `task`，确认 `task_index -> tasks.parquet.index` 映射是否未被改动。
3. **动作标签健康度**
   - 先抽样检查 `action` 是否仍为全零；
   - 若全零，训练“可收敛但不可用”是预期现象，不是训练器 bug。
4. **机器人元信息**
   - 检查是否有流程要求 `robot_type` 或 `action/state names` 非空。
5. **统计项一致性**
   - 对 string feature 的 stats 缺失可保留为告警，但需确认不会被你的训练前处理当作 fatal。

## 11. 审计最终结论（架构师裁决）

### 11.1 数据是否“坏掉/会崩”

- 结论：**不会因索引与结构坏掉而崩**。
- 证据：A 的目录、索引链路、任务映射、视频引用、时间戳边界全部通过。

### 11.2 数据是否“可直接按黄金参考同配置训练”

- 结论：**不建议直接同配置训练**。
- 原因：
  - schema 不同（相机键、语言字段、environment_state、robot_type、names）；
  - A 的 `action` 全零会导致策略学习退化。

### 11.3 这份数据最可能引发的问题

1. 训练不报错但策略输出接近全零；
2. 复用 B 的配置时报 feature key mismatch；
3. 依赖 `robot_type` 或 `names` 的路径发生兼容性异常；
4. 质量检查长期提示 `language_instruction` stats 缺失（告警级，不是致命级）。
