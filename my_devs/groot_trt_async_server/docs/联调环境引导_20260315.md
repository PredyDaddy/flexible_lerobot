# GR00T TRT Async Server 联调环境引导

更新时间：2026-03-15

本文档只覆盖 `my_devs/groot_trt_async_server/` 这条开发线的环境与启动前核验，不处理模型正确性、协议正确性和真机联调结论。

## 1. 目标

在宣布“进入联调准备”之前，至少要先回答三个问题：

1. 当前命令是不是运行在 `lerobot_flex` 环境里
2. `grpc` 和 async client/server 入口是不是可导入
3. server 启动前的最小 smoke 路径是不是已经打通

本目录下新增的检查入口是：

- `my_devs/groot_trt_async_server/check_env.py`

## 2. 最小可运行依赖清单

针对当前 `my_devs/groot_trt_async_server/` 代码路径，最小运行依赖至少包括：

- `lerobot_flex` conda 环境
- `torch`
- `grpcio`
- 当前仓库本身可被 Python 导入
- `my_devs.groot_trt_async_server.policy_server`
- `my_devs.groot_trt_async_server.robot_client`

说明：

- `grpcio-tools` 不是当前运行时硬门槛，本引导不把它列为启动前必需项。
- 真正的 robot/camera 设备依赖只在真实 client 或半实物 roundtrip 时才是硬门槛；纯环境核验不要求设备在线。

## 3. 推荐核验命令

所有命令都必须使用 `lerobot_flex` 环境。

### 3.1 全量环境检查

```bash
/home/cqy/miniconda3/bin/conda run -n lerobot_flex \
  python my_devs/groot_trt_async_server/check_env.py --mode all
```

用途：

- 校验当前 Python 是否落在 `lerobot_flex`
- 校验 `torch`、`grpc`、server/client 入口 import
- 执行一次 `run_server.py --dry-run`

### 3.2 只检查 server 侧

```bash
/home/cqy/miniconda3/bin/conda run -n lerobot_flex \
  python my_devs/groot_trt_async_server/check_env.py --mode server
```

适用场景：

- 环境刚补完 `grpcio`
- 只想确认 server import 和 `--dry-run` 路径

### 3.3 只检查 client 侧

```bash
/home/cqy/miniconda3/bin/conda run -n lerobot_flex \
  python my_devs/groot_trt_async_server/check_env.py --mode client --skip-server-dry-run
```

适用场景：

- 只确认 async client 依赖是否齐全
- 不想重复执行 server dry-run

## 4. 输出判读

脚本输出每一项检查的 `PASS` 或 `FAIL`。

最关键的几项是：

- `conda-env`
- `import:grpc`
- `import:my_devs.groot_trt_async_server.policy_server`
- `robot-client-async-stack`
- `server-dry-run`

判读原则：

- 只要 `import:grpc` 失败，就仍然处于环境硬阻塞状态
- 只要 `server-dry-run` 失败，就不能把 server 启动前 smoke 视为已通过
- 只要 `robot-client-async-stack` 失败，就不能把 client async 入口视为已就绪

## 5. 当前已知事实

截至 2026-03-15，本仓库在 `lerobot_flex` 环境中的已知事实是：

- `grpc` 当前缺失
- `run_server.py --dry-run` 会因此失败

所以当前状态仍然是：

- `环境阻塞未解除`
- `不可进入联调准备`

后续这两项是否关闭，应以 `check_env.py` 的实际输出为准，而不是以口头说明或历史文档为准。

## 6. 联调前最低通过线

在环境维度，至少同时满足下面三条，才允许把结果提交给下一轮准入复核：

1. `check_env.py --mode all` 退出码为 `0`
2. `server-dry-run` 为 `PASS`
3. `robot-client-async-stack` 为 `PASS`

如果以上任一条件不成立，当前结论只能是：

- `环境与启动前核验未通过`

## 7. 建议执行顺序

建议按这个顺序做：

1. 先运行 `check_env.py --mode all`
2. 如果 `grpc` 缺失，先补环境，再重复运行
3. `server-dry-run` 通过后，再继续做 protocol/control-plane 整改验证
4. 最后再进入真实 roundtrip 或准入复核
