## 2. 本轮复核后的真实状态

### 2.1 已确认通过的事项

- `run_mock_roundtrip.py --pure-mock` 当前可执行通过
- `check_groot_trt_backend.py` 当前可执行通过
- `check_env.py --mode all` 已完成环境核验
- server dry-run 可运行并能暴露配置/导入问题

### 2.2 已确认仍然阻断的事项

#### 阻断 A: 控制面失败终态不闭环

独立复核指出：

- 当前协议只覆盖正向 ack，没有覆盖失败终态
- stale observation、session 切换后丢弃 chunk、服务端异常等路径没有把失败绑定回 `request_id/session_id`
- 客户端 inflight request 可能永久卡死
- 当前 pure mock 主要覆盖 helper，不足以证明真实协议闭环

架构结论：

- 控制面主路径虽已明显收敛，但失败路径闭环仍未完成

#### 阻断 B: 后端自检存在假绿风险

独立复核指出：

- 现在只能证明锁内调用过一次 sync
- 不能证明“最后一次 TRT 工作后再 unlock”
- session 工作事件没有进入统一 timeline
- sync 的 stream 与执行工作 stream 没有建立可验证绑定
- CLI 输出对当前自检能力表述过强

架构结论：

- 后端自检已经从脆弱字符串检查升级到了初步行为检查
- 但还不能作为准入级证据使用

#### 阻断 C: 外部依赖未补齐

环境核验指出：

- `lerobot_flex` 中缺少 `grpc`

架构结论：

- 当前仍不能做真实 gRPC 端到端准入验证
