# GR00T TRT Async Server 最小协议补丁任务单

更新时间：2026-03-15

本文档只服务于最小协议闭环整改，不处理资源所有权、sticky session 或环境依赖问题。

## 1. 目标

本任务只解决三个最小问题：

1. `request_id` 必须进入 observation/action 传输载荷
2. server 返回 action 时必须带回与 observation 对应的 `request_id` 或等价 ack 信息
3. client 必须显式丢弃旧 chunk、晚到 chunk、重复 chunk

附加目标：

- `run_mock_roundtrip.py` 提供纯协议级 mock 模式，不依赖真实 robot/camera

## 3. 最小实现要求

### 3.1 request_id 过协议

- observation 发出时带 `request_id`
- server 处理后返回 action chunk 时带同一个 `request_id`
- client 不再只靠本地 `_pending_request` 推断请求关联

### 3.2 chunk 显式丢弃规则

client 至少要有下面三类显式处理：

- 旧 `request_id` 的 chunk 直接丢弃
- 当前 `request_id` 但 `timestep` 早于已执行上界的 chunk 直接丢弃
- 同一 `request_id + timestep` 的重复 chunk 直接丢弃

### 3.3 纯协议 mock

- `run_mock_roundtrip.py` 能在不连接真实 robot/camera 的情况下运行
- mock 模式只验证 observation -> request_id -> action chunk 的协议往返
- 不允许把半实物路径继续包装成纯 mock

## 4. 验收标准

最小通过标准：

1. `request_id` 在 client 发出和 server 回回来的两个方向都可见
2. client 代码里存在显式丢弃旧/晚到/重复 chunk 的逻辑
3. `run_mock_roundtrip.py` 具备纯 mock 开关或纯 mock 默认路径
4. 在 `lerobot_flex` 中至少完成 `py_compile` 或等价 smoke
