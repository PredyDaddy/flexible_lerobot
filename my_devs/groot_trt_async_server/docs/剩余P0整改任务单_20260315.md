## 3. 剩余 P0 微任务

### 微任务 C：server 资源所有权收口

目标：

- server 不再直接把 client 下发的本地路径当作最终加载决策
- client 只能传递 server 可接受的逻辑资源标识、资源 profile 或等价描述
- server 侧负责把逻辑标识解析为本地 checkpoint / engine / processor 资源

唯一写入边界：

- `my_devs/groot_trt_async_server/configs.py`
- `my_devs/groot_trt_async_server/policy_server.py`
- `my_devs/groot_trt_async_server/run_server.py`
- 如确有必要，可最小改动：
  - `my_devs/groot_trt_async_server/robot_client.py`
  - `my_devs/groot_trt_async_server/run_client.py`

明确不做：

- 不处理 sticky session 恢复模型
- 不处理 `grpc` 环境依赖
- 不扩散到 backend 文件

必须交付的最小结果：

1. client 不再直接控制 server 本地 checkpoint / engine 路径
2. server 端存在显式可审计的资源解析入口
3. 错误资源标识会显式失败，而不是静默回退

最低验证要求：

- `py_compile` 通过
- 至少一条本地 smoke 命令能证明：
  - server 资源解析入口存在
  - 非法资源标识会失败

### 微任务 D：sticky session 恢复模型

目标：

- 把 sticky session 从“只会拒绝别的 peer”升级为最小可恢复模型
- 至少明确一种可恢复路径：
  - release
  - reconnect
  - safe takeover

唯一写入边界：

- `my_devs/groot_trt_async_server/policy_server.py`
- 如确有必要，可最小改动：
  - `my_devs/groot_trt_async_server/robot_client.py`

明确不做：

- 不处理资源所有权
- 不处理 `grpc` 环境依赖
- 不扩散到 backend 文件

必须交付的最小结果：

1. session 绑定不再只依赖 `context.peer()`
2. 至少一条会话恢复路径有明确实现
3. 对冲突 session 的行为是显式、可审计的

最低验证要求：

- `py_compile` 通过
- 至少一个本地 mock / 纯逻辑自检入口能覆盖会话 claim / release / reconnect / takeover 中的一条路径

### 微任务 E：环境阻塞解除

目标：

- 解除 `grpc` 环境阻塞，使 `check_env.py --mode all` 至少具备通过条件

说明：

- 这是运行环境任务，不是代码结构任务
- 是否真正执行安装动作，可在前两个微任务完成后再决定

## 4. 执行顺序
