# GR00T TRT Async Server 最小 TRT 自检任务单

更新时间：2026-03-15

本文档只服务于 TRT backend 的最小验证闭环，不处理模型导出、数值对齐或真机推理问题。

## 1. 目标

本任务只解决两件事：

1. 不回退已有的串行化保护、token 一致性校验、stream 同步
2. 把这些保护从“口头结论”变成“可执行自检”

## 2. 写入边界

只允许修改：

- `my_devs/groot_trt_async_server/groot_trt_policy.py`
- 新增 `my_devs/groot_trt_async_server/` 下的轻量验证脚本

禁止修改：

- `run_groot_infer_trt.py`
- 任何 `my_devs/groot_trt_async_server/` 之外的文件

## 3. 最小实现要求

### 3.1 自检入口

至少落一种：

- 独立验证脚本
- 模块级自检函数
- 可触发的轻量断言入口

### 3.2 自检至少覆盖

- 共享 session 保护已启用
- image-token / ViT token 一致性校验会显式失败
- stream 同步路径可被触发或可被检查

### 3.3 不允许的做法

- 不允许大改模型逻辑
- 不允许把未验证项写成“已验证”
- 不允许要求真实 gRPC 环境或真机依赖才能跑最小自检

## 4. 验收标准

最小通过标准：

1. 新增了可执行自检入口
2. 在 `lerobot_flex` 至少完成 `py_compile` 或脚本执行
3. 回报中明确区分“已验证”和“仍待联调验证”

## 5. 回报模板

只允许按下面格式回报：

1. `Implemented`
2. `Verification`
3. `Residual Risks`
4. `Files Changed`
5. `Ready For Re-Review`
