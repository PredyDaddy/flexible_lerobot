# 数据检查与训练脚本工作报告

日期：2026-02-27  
位置：`my_devs/train/act`

## 1. 工作目标

- 基于当前仓库 `flexible_lerobot`，提供两套 ACT 训练脚本：
- `train_smoke.sh`：用于训练管线快速验证（smoke test）。
- `train_full.sh`：用于长时训练模板（接近旧脚本风格）。
- 不依赖机器人注册，优先支持本地离线数据集。

## 2. 我做了什么

- 新建脚本：
  - `my_devs/train/act/train_smoke.sh`
  - `my_devs/train/act/train_full.sh`
- 两个脚本都支持：
  - `conda run -n lerobot_flex`（若当前不在该环境）
  - 本地数据集路径与 repo_id 可覆盖
  - 设备自动选择（`auto -> cuda/cpu`）
  - 日志自动落盘到 `logs/`
  - `DRY_RUN=1` 预演命令

## 3. 脚本默认策略

- `train_smoke.sh` 默认参数：
  - `steps=10`、`batch_size=1`、`num_workers=0`、`eval_freq=-1`
  - 目标是快速判断“训练链路是否可跑通”。
- `train_full.sh` 默认参数：
  - `steps=100000`、`batch_size=32`、`save_freq=10000`、`log_freq=100`
  - `eval_freq` 默认仍为 `-1`（当前仓库未注册机器人，避免评估路径中断）。
  - 如后续评估环境就绪，可改为 `EVAL_FREQ=10000`。

## 4. 验证结果

- 已完成语法与 dry-run 检查（通过）。
- 已做真实烟雾训练验证（`STEPS=1`），训练成功结束，退出码 `0`。
- 结论：当前脚本可用于“训练管线测试”目标。

## 5. 已知注意点

- 离线模式下，若某些权重未缓存，可能触发下载失败。
- 数据集若使用 AV1 编码，依赖本机解码能力。
- 建议先用 `train_smoke.sh` 通过后，再跑 `train_full.sh`。
